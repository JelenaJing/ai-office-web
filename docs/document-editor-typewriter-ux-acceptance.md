# Document Editor Typewriter UX — 验收文档

版本：v1.1  分支：feat/web-feature-parity

## 变更说明

将之前的"块级流式写入"（block streaming）升级为"字符级打字机效果"（typewriter）：

- 后端不变：仍一次性返回完整 HTML。
- 前端新增 `typewriterDocument.ts`，按字符逐步写入编辑器，模拟 AI 打字效果。
- 不使用 SSE / WebSocket。

| 文件 | 说明 |
|------|------|

## 验收项

### 基础打字机效果

- [ ] 输入生成需求，点击"生成初稿"，后端返回后正文出现打字机效果（逐字出现）
- [ ] 不是一次性整篇出现
- [ ] 打字速度合理（长文 ≤ 22 秒完成前端打字）
- [ ] 标点后有停顿，段落间有节奏感
- [ ] 编辑器自动滚动到当前正在写入的位置

### 停止生成

- [ ] 打字机运行期间，"生成初稿"按钮变为"停止生成"（红色）
- [ ] 点击"停止生成"后，打字立即停止
- [ ] 已打出内容保留在编辑器中，不被清空
- [ ] 右侧状态栏显示"已停止生成，已写入内容已保留"

### 状态显示

- [ ] 点击"生成初稿"后，状态显示"AI 正在构思…"（等待后端返回期间）
- [ ] 后端返回、打字开始后，状态切换为"AI 正在写入正文…"
- [ ] 实时显示"AI 正在写入… 已完成 x / y 字"
- [ ] 完成后状态切换为"初稿已生成，可继续修改或下载"

### 完成后 ActionCard

- [ ] 打字完成后右侧出现操作卡片，包含：
  - "下载 Word"
  - "下载 Markdown"
  - "下载 HTML"
  - "继续修改"
- [ ] 停止生成后，操作卡片显示"已完成：停止生成"

### 导出

- [ ] 完成打字后，点击"下载 Word" → 文件自动下载
- [ ] Word 内容包含当前编辑器最新内容（非空）
- [ ] 点击"下载 Markdown" → 文件自动下载
- [ ] 下载无需去资源中心
- [ ] 点击"下载 HTML" → 文件自动下载
- [ ] PDF 显示明确降级提示，不静默失败

### 格式一致性

- [ ] 打字完成后，导出格式与原始 AI 生成 HTML 一致（标题/列表/加粗正确）
- [ ] 打字过程中看到的段落格式在完成后仍然保留
- [ ] session.html 与编辑器内容一致（完成时同步）

### 草稿持久化

- [ ] 生成/编辑完成后保存到 localStorage (`web-document-draft:${workspacePath}`)
- [ ] 刷新页面后自动恢复最近草稿
- [ ] 顶部显示"草稿已恢复"提示

### 不允许事项

- [ ] ❌ 不允许打字机运行期间页面卡死
- [ ] ❌ 不允许取消后清空已打出内容
- [ ] ❌ 不允许下载静默失败
- [ ] ❌ 不允许 PDF 下载假成功

## 技术参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| typingSpeed | 15ms/char | 每字符基准速度 |
| sentencePauseMs | 120ms | 标点后额外停顿 |
| blockPauseMs | 250ms | 段落间停顿 |
| batchSize | 1-4 chars | 高速时批量写入 |
| 最大时长 | 22 秒 | 超出时自动加速 |
