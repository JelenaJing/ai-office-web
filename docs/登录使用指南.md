# AI-Office 登录使用指南

## 概述

AI-Office 使用内部账号中心（AccountCenter）进行身份认证。您需要使用机构分配的用户名和密码完成登录，系统将在登录后自动配置内部邮件、即时通讯等关联服务，无需额外操作。

---

## 一、登录步骤

### 第一步：打开设置面板，找到账号入口

启动 AI-Office 后，打开**设置**面板，找到标题为 **🔐 内部账号（AccountCenter）** 的区域，即可看到登录表单。

### 第二步：填写登录信息

| 字段 | 说明 |
|------|------|
| **用户名** | 机构分配的账号名（输入时会自动去除首尾空格） |
| **密码** | 账号对应的密码（输入时内容不可见） |

> **注意：** 用户名和密码均不能为空，两者都填写后登录按钮才会变为可点击状态。

### 第三步：点击"登录内部账号"

点击蓝色的 **「登录内部账号」** 按钮，按钮将变为 **「登录中…」** 并进入禁用状态，请等待系统完成身份验证（超时上限为 10 秒）。

### 第四步：登录成功

验证通过后，登录表单消失，界面切换为账号信息面板，显示以下内容：

| 字段 | 说明 |
|------|------|
| 用户名 | AccountCenter 登录名 |
| 显示名 | 用于邮件署名和聊天显示的名称 |
| 邮箱 | 机构内部邮箱地址 |
| 角色 | 账号权限角色（如 `admin`、`super_admin` 等） |
| AccountCenter 状态 | `active`（正常）或 `disabled`（已禁用） |

账号面板下方还会显示三项**服务绑定状态**（mail / matrix / office），以及 **📧 内部邮箱** 和 **💬 内部通讯** 两个功能区。

### 第五步：后台自动配置（无需手动操作）

登录成功后，系统将在后台自动完成以下操作：

- **内部邮件**：自动写入 IMAP/SMTP 配置，邮件客户端将立即可用。
- **服务绑定**：自动拉取 mail / matrix / office 三项服务的绑定与同步状态。
- **即时通讯**：自动恢复 Matrix 聊天会话（如已绑定）。

> 以上操作均为非阻塞后台任务，不会导致登录等待。

---

## 二、登录成功后的功能区说明

### 📧 内部邮箱

登录后，设置面板中会出现**内部邮箱**区域，包含以下信息：

| 项目 | 说明 |
|------|------|
| 邮箱地址 | 您的机构邮箱（如 `username@ai.cuhk.edu.cn`） |
| Webmail | 点击「**打开 SOGo**」在浏览器中访问网页版邮件 |
| 配置状态 | 显示自动配置是否成功（见下方状态说明） |

**邮件自动配置状态说明：**

| 状态提示 | 含义 |
|----------|------|
| `正在自动配置内部邮箱...` | 配置写入中，请稍候 |
| `✓ 内部邮箱已自动配置` | 配置成功，邮件客户端即刻可用 |
| `邮箱自动配置失败：…` | 写入出错，可点击「**应用内部邮箱配置**」手动重试 |

> **提示：** 邮箱的登录密码与 AccountCenter 密码相同。如果您此前设置过自定义邮箱，内部邮箱配置写入后原有配置将被替换。

### 💬 内部通讯

登录后，**内部通讯**区域显示聊天入口：

| 项目 | 说明 |
|------|------|
| 服务 | AccountCenter Chat，支持单聊与群聊 |
| 操作 | 点击「**打开内部通讯**」即可进入聊天窗口 |

> 也可在界面底部状态栏点击 **💬 通讯** 按钮，或在左侧边栏切换到**通讯**标签页，直接打开聊天窗口。

## 三、特殊场景处理

### 首次登录 / 需要修改初始密码

如果管理员为您设置了初始密码，登录后账号面板顶部会出现黄色警告：

> ⚠ 请尽快修改初始密码

**修改密码步骤：**

1. 在设置面板中找到**修改密码**功能
2. 输入**当前密码**（即初始密码）
3. 输入**新密码**并确认
4. 提交后警告提示将自动消失

### 账号已被禁用

登录时若看到以下红色错误提示：

> 该内部账号已被禁用，请联系管理员

您的账号已被 IT 管理员停用。请联系机构管理员恢复账号权限后再重新登录。

### 服务绑定读取失败

登录成功后，如果绑定状态区域出现黄色提示：

> 服务绑定状态读取失败

这**不影响**账号的正常登录与使用，仅表示服务状态信息暂时无法获取。可点击 **「刷新绑定状态」** 按钮手动重试。

## 四、自动会话恢复（免重复登录）

AI-Office 在退出后会安全保存登录凭证（Token）。下次启动应用时，系统将自动验证并恢复登录状态，无需重新输入密码。

| 情况 | 结果 |
|------|------|
| Token 有效 | 直接进入已登录状态，服务自动恢复 |
| Token 过期或失效 | 自动清除凭证，返回登录表单，需重新登录 |

> **注意：** 会话恢复时密码不会从磁盘读取（密码从不落盘）。应用重启后若需重新应用邮箱配置，请退出并重新登录，系统会在登录时自动重新配置。

---

## 五、退出登录

在账号面板底部，点击红色的 **「退出登录」** 按钮即可安全退出。退出后：

- 本地保存的 Token 文件将被立即删除
- 内存中的密码将被销毁
- 再次打开应用需要重新输入用户名和密码

## 六、常见问题

**Q：登录按钮不可点击？**  
A：用户名和密码两个字段都必须填写，任一为空时登录按钮处于禁用状态。

**Q：登录时提示网络错误或请求超时？**  
A：AccountCenter 服务仅限机构内网访问，请确认您的设备已连接到内部网络（VPN 或办公网），无法从外网直接登录。

**Q：输入正确的用户名密码但仍提示错误？**  
A：请检查用户名是否含有全角字符或中间空格（系统只会自动去除首尾空格）。如仍无法登录，请联系管理员确认账号状态。

**Q：密码会保存到本地磁盘吗？**  
A：不会。密码仅在当次运行期间保留于内存，应用关闭后立即销毁，不写入任何本地文件。

**Q：登录后邮件没有自动配置成功？**  
A：在 **📧 内部邮箱** 区域点击 **「应用内部邮箱配置」** 手动重试。如果应用重启后提示需要重新登录，请退出后重新登录一次，系统会在登录时自动重新配置。

**Q：内部通讯无法打开？**  
A：请确认已登录内部账号，并检查内网连接是否正常。如仍无法打开，请联系管理员确认 AccountCenter Chat 服务是否可用。

**Q：我的邮件客户端原有配置被覆盖了？**  
A：内部邮箱配置会替换原有的自定义邮件配置。如需保留自定义配置，请在**邮件设置**中重新手动添加。
